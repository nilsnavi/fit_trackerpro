# Active Workout: одноэкранный поток (2026-04)

## Что изменилось

- **Компактная шапка сессии** (`ActiveWorkoutSessionBar`): название, прошедшее время, прогресс подходов и синк-статус в одном блоке без дублирования карточки «Сейчас».
- **Параметры сессии** (длительность, комментарий, инсайты, пресеты отдыха) вынесены в **сворачиваемый блок** (`ActiveWorkoutSessionDetailsCollapsible`), чтобы по умолчанию не отвлекать от ввода подходов.
- **Горизонтальная навигация по упражнениям** (`ActiveExerciseNavigator`) и кнопки «Пред./след. упр.» — переход между упражнениями без смены маршрута, только смена позиции в Zustand/React Query.
- **Таймер отдыха**: инлайн-панель убрана; управление перенесено в **`FloatingRestTimer`** (пауза, сброс, пропуск) — не блокирует скролл и ввод.
- **Быстрый старт отдыха** и **редактирование пресетов** доступны из шапки и из свёрнутого блока параметров (раньше — в большой карточке «Сейчас»).
- **Список упражнений**: отключено авто-сворачивание неактивных блоков — видны все упражнения; сворачивание вручную по кнопке сохранено.
- **Нижний rail**: убран дублирующий прогресс «X/Y подходов» (он в шапке); остались пресеты отдыха и действия «+упражнение/таймер», «± подход», «Завершить».

## Что удалено

- `CurrentExerciseCard.tsx` — дублировала прогресс, отдых и CTA «Готово», которые уже есть в строках подходов (`ExerciseSetRow`).
- `RestTimerPanel.tsx` — дублировала тикание таймера с плавающим виджетом.
- `useActiveWorkoutContextCard.ts` — модель только для удалённой карточки.

## Как это снижает трение

- Меньше вертикального скролла между «контекстом сессии» и подходами; один визуальный якорь (шапка + список).
- Нет второго «экрана в экране» в виде крупной карточки текущего упражнения.
- Переключение упражнений — один тап по чипу или по «След. упр.» без навигации роутера.
- Оптимистические обновления подходов **без изменений**: по-прежнему `useOptimisticWorkoutSession` + `patchItem`; добавление подхода копирует последние вес/повторы там же, где и раньше.
