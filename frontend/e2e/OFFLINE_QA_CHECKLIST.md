# Offline/Retry UX — Manual QA Checklist

Чеклист для ручной проверки offline-поведения active workout flow.
Открыть DevTools → Network → Offline для симуляции потери сети.

## 1. Старт тренировки офлайн
- [ ] Включить offline
- [ ] Нажать "Начать тренировку"
- [ ] Ожидание: toast "Старт тренировки в очереди — отправим при восстановлении сети"
- [ ] Навигация корректна (не зависает)
- [ ] Включить online → тренировка создаётся на сервере

## 2. Отметка подхода офлайн
- [ ] Начать тренировку (online), затем переключить в offline
- [ ] Нажать "Готово" на подходе
- [ ] Ожидание: SyncStateDot показывает синюю точку "Сохраняется..."
- [ ] Через 2с: SyncStateDot → жёлтая точка "Офлайн"
- [ ] Toast "Офлайн: изменения поставлены в очередь синхронизации" (один раз)
- [ ] Данные подхода (вес, повторения) сохраняются в localStorage

## 3. Восстановление сети (reconnect)
- [ ] Продолжить из п.2: включить online
- [ ] Ожидание: SyncStateDot → "Синк..." → исчезает (synced)
- [ ] Toast "Синхронизация восстановлена, изменения сохранены"
- [ ] WorkoutSyncQueueStatus зелёный → через 3с скрывается

## 4. Завершение тренировки офлайн
- [ ] Начать тренировку online, отметить подходы
- [ ] Переключить в offline
- [ ] Нажать "Завершить тренировку"
- [ ] FinishWorkoutModal показывает warning: "Вы офлайн — результат будет отправлен при восстановлении сети"
- [ ] Подтвердить завершение
- [ ] Ожидание: НЕТ error toast, вместо этого: "Тренировка завершена — результат отправится при восстановлении сети"
- [ ] Навигация на /workouts
- [ ] Включить online → тренировка синхронизируется

## 5. Удаление упражнения офлайн
- [ ] Находясь offline, удалить упражнение из тренировки
- [ ] Ожидание: toast "Упражнение удалено (сохранено локально)"
- [ ] Упражнение визуально удалено из списка
- [ ] Включить online → синхронизируется

## 6. Partial sync failure
- [ ] Имитировать: включить offline → внести изменения → включить online → быстро offline
- [ ] Проверить: WorkoutSyncQueueStatus показывает count ошибок
- [ ] Кнопка "Повторить все" работает
- [ ] Retry отдельного item работает

## 7. Duplicate submit protection
- [ ] Нажать "Завершить тренировку" → быстро нажать дважды "Подтвердить"
- [ ] Ожидание: кнопка блокируется (disabled + isLoading) после первого клика
- [ ] Только одна мутация отправляется

## 8. FinishWorkoutModal sync-state при error
- [ ] Имитировать ошибку синхронизации (throttle network)
- [ ] Открыть FinishWorkoutModal
- [ ] Ожидание: warning "Не удалось синхронизировать последние изменения..."
- [ ] Подтвердить завершение — данные ставятся в очередь без потери

## 9. WorkoutSyncQueueStatus auto-collapse
- [ ] При нормальном online-режиме: бадж "Синхронизировано" → через 3с скрывается
- [ ] При появлении pending/failed: бадж сразу виден
- [ ] При переключении offline → бадж сразу показывает "Нет сети"

## 10. Полный offline flow (от start до finish)
- [ ] Включить offline
- [ ] Начать тренировку → toast
- [ ] Добавить упражнения
- [ ] Отметить подходы
- [ ] Удалить упражнение → toast с "(сохранено локально)"
- [ ] Завершить тренировку → warning в модалке → info toast
- [ ] Включить online
- [ ] Проверить: все данные синхронизировались, тренировка в истории
